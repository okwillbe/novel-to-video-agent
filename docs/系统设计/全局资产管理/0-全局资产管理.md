# 资产管理模块设计文档

> 每个用户都有自己的全局资产和项目资产。全局资产可以通过文件夹进行组织。

## 1. 资产类型概览

```
全局资产中心
│
├── 📂 GlobalAssetFolder (文件夹) - 组织单元
│
├── 👤 GlobalCharacter (角色)
│   └── 🖼️ GlobalCharacterAppearance (角色形象，子资产)
│
├── 🏠 GlobalLocation (场景)
│   └── 🖼️ GlobalLocationImage (场景图片，子资产)
│
└── 🎙️ GlobalVoice (音色)
```

---

## 2. 数据模型设计

### 2.1 GlobalAssetFolder（文件夹）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 自增主键 |
| folder_id | VARCHAR(64) | UUID，业务主键 |
| user_id | VARCHAR(64) | 所属用户 |
| name | VARCHAR(128) | 文件夹名称 |
| sort_order | INT | 排序 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

---

### 2.2 GlobalCharacter（角色）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 自增主键 |
| character_id | VARCHAR(64) | UUID |
| user_id | VARCHAR(64) | 所属用户 |

| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## 4. REST API 设计

### 4.1 文件夹 API

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/v1/assets/folders` | GET | 列出文件夹 |
| `/api/v1/assets/folders` | POST | 创建文件夹 |
| `/api/v1/assets/folders/{folderId}` | DELETE | 删除文件夹 |

### 4.2 角色 API

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/v1/assets/characters` | GET | 列出角色（可选 folderId 参数） |
| `/api/v1/assets/characters/{characterId}` | GET | 获取角色详情 |
| `/api/v1/assets/characters` | POST | 创建角色 |
| `/api/v1/assets/characters/{characterId}` | PUT | 更新角色 |
| `/api/v1/assets/characters/{characterId}` | DELETE | 删除角色 |
| `/api/v1/assets/characters/{characterId}/appearances` | GET | 列出角色形象 |
| `/api/v1/assets/characters/{characterId}/appearances` | POST | 添加角色形象 |
| `/api/v1/assets/appearances/{appearanceId}` | DELETE | 删除角色形象 |

### 4.3 场景 API

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/v1/assets/locations` | GET | 列出场景 |
| `/api/v1/assets/locations/{locationId}` | GET | 获取场景详情 |
| `/api/v1/assets/locations` | POST | 创建场景 |
| `/api/v1/assets/locations/{locationId}` | PUT | 更新场景 |
| `/api/v1/assets/locations/{locationId}` | DELETE | 删除场景 |
| `/api/v1/assets/locations/{locationId}/images` | GET | 列出场景图片 |
| `/api/v1/assets/locations/{locationId}/images` | POST | 添加场景图片 |
| `/api/v1/assets/location-images/{imageId}` | DELETE | 删除场景图片 |

### 4.4 音色 API

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/v1/assets/voices` | GET | 列出音色 |
| `/api/v1/assets/voices/{voiceId}` | GET | 获取音色详情 |
| `/api/v1/assets/voices` | POST | 创建音色 |
| `/api/v1/assets/voices/{voiceId}` | PUT | 更新音色 |
| `/api/v1/assets/voices/{voiceId}` | DELETE | 删除音色 |

## 5. 前端页面设计

### 5.1 页面结构

```
┌─────────────────────────────────────────────────────────┐
│ 资产管理                                                │
├──────────────┬──────────────────────────────────────────┤
│ 文件夹       │  [角色] [场景] [音色]  ← Tabs            │
│              │                                          │
│ 📂 全部资产  │  ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│ 📂 角色集    │  │ 角色卡片 │ │ 角色卡片 │ │ 角色卡片 │   │
│ 📂 场景集    │  └─────────┘ └─────────┘ └─────────┘   │
│ 📂 音色集    │                                          │
│              │  ┌─────────┐ ┌─────────┐               │
│ [+ 新建]     │  │ 角色卡片 │ │ 角色卡片 │               │
│              │  └─────────┘ └─────────┘               │
└──────────────┴──────────────────────────────────────────┘
```

### 5.2 功能特性

- **左侧**：文件夹树形导航，支持创建/删除文件夹
- **右侧**：资产列表（Tabs 切换：角色/场景/音色）
- **筛选**：点击文件夹可筛选该文件夹下的资产
- **操作**：每个资产卡片支持编辑/删除操作

## 7. 使用说明

### 7.1 初始化数据库

```bash
mysql -h 127.0.0.1 -P 13306 -u root -pnovel2video123 novel2video < sql/init-assets.sql
```

### 7.2 启动服务

```bash
# 后端
mvn spring-boot:run -pl agent-web

# 前端
cd frontend && npm run dev
```

### 7.3 访问页面

- 资产管理页面：http://localhost:5173/assets
- API 文档：http://localhost:8080/swagger-ui.html

## 8. 总结

| 资产类型 | 说明 | 子资产 |
|----------|------|--------|
| GlobalAssetFolder | 文件夹（组织单元） | - |
| GlobalCharacter | 角色 | GlobalCharacterAppearance（多形象） |
| GlobalLocation | 场景 | GlobalLocationImage（多图片） |
| GlobalVoice | 音色 | - |

共 3 大资产类型，角色和场景各有自己的子资产（形象/图片）。所有资产都可以选择性地归属到文件夹进行组织管理。
