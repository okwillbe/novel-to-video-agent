1. 概述

角色形象是角色的子资产，用于管理同一角色的不同外观状态（如便装、战甲、受伤等）。

┌──────────┬────────────────────────────────────────────┐
│   特性   │                    说明                    │
├──────────┼────────────────────────────────────────────┤
│ 所属关系 │ 属于角色（GlobalCharacter），1:N           │
├──────────┼────────────────────────────────────────────┤
│ 核心功能 │ 多版本图片管理 + 描述词管理 + AI 生成/修改 │
├──────────┼────────────────────────────────────────────┤
│ 设计目的 │ 支持角色在不同场景/状态下的外观变化        │
├──────────┼────────────────────────────────────────────┤
│ 命名含义 │ "形象" = 角色的一个外观版本                │
└──────────┴────────────────────────────────────────────┘

2. 数据模型

prisma
model GlobalCharacterAppearance {
id           String   @id @default(uuid())
characterId  String   // 所属角色

    // ========== 形象标识 ==========
    appearanceIndex Int           // 形象序号（0=默认）
    changeReason  String          // 形象说明（如"便装"、"战甲"）

    // ========== 风格与描述 ==========
    artStyle        String?       // 艺术风格
    description     String?       // 当前使用的描述词
    descriptions    String?       // 历史描述词列表（JSON数组）
    descriptionSource String?     // 描述来源：'user' | 'ai'

    // ========== 当前图片 ==========
    imageUrl        String?       // 当前显示的图片URL（单张）
    imageMediaId    String?       // 当前图片媒体ID
    imageMedia      MediaObject?  @relation(...)

    // ========== 候选图片 ==========
    imageUrls       String?       // 候选图片列表（JSON数组）
    imageMedias     MediaRef[]?   // 候选图片媒体列表
    selectedIndex   Int?          // 当前选中的候选图索引

    // ========== 撤销支持 ==========
    previousImageUrl     String?  // 上一次的图片URL
    previousImageMediaId String?
    previousImageUrls    String?  // 上一次的候选列表
    previousDescriptions String?  // 上一次的描述列表

    // ========== 时间戳 ==========
    createdAt DateTime @default(now())
    updatedAt DateTime @updatedAt

    @@unique([characterId, appearanceIndex])
    @@index([characterId])
    @@map("global_character_appearances")
}

6. 接口详情

6.1 创建形象

// POST /api/asset-hub/appearances
// Request
{
characterId: string            // 必填：所属角色
appearanceIndex?: number       // 形象序号，自动递增
changeReason: string           // 必填：形象说明
artStyle?: string              // 艺术风格
description?: string           // 描述词
initialImageUrl?: string       // 初始图片URL
}

// Response
{
success: true,
appearance: GlobalCharacterAppearance
}

6.2 更新形象

// PATCH /api/asset-hub/characters/[characterId]/appearances/[appearanceIndex]
// Request（所有字段可选）
{
changeReason?: string          // 更新形象说明
description?: string           // 更新描述词
artStyle?: string              // 更新风格
selectedIndex?: number         // 切换选中的图片
}

6.3 生成形象图片

// POST /api/asset-hub/generate-image
// Request
{
type: 'character'              // 必填
id: string                     // 角色ID
appearanceIndex: number        // 形象序号
artStyle?: string              // 艺术风格（可选）
count?: number                 // 生成数量，默认 4
}

// Response
{
taskId: string,
status: 'queued'
}

// 图片生成后：
// 1. 图片存入 appearance.imageUrls
// 2. selectedIndex 默认为 0
// 3. 通过 SSE 推送进度

6.4 修改单张图片

// POST /api/asset-hub/modify-image
// Request
{
type: 'character'
id: string                     // 角色ID
appearanceIndex: number        // 形象序号
imageIndex: number             // 要修改的图片索引
modifyPrompt: string           // 修改指令（如"把衣服改成蓝色"）
extraImageUrls?: string[]      // 参考图（可选）
}

// 行为：
// 1. 保存当前状态到 previous* 字段
// 2. 调用 AI 局部重绘
// 3. 替换 imageUrls[imageIndex]

6.5 选择候选图片

// POST /api/asset-hub/select-image
// Request
{
type: 'character'
id: string
appearanceIndex: number
imageIndex: number             // 选中的图片索引
}

// 行为：更新 selectedIndex

6.6 撤销图片修改

// POST /api/asset-hub/undo-image
// Request
{
type: 'character'
id: string
appearanceIndex: number
imageIndex?: number            // 可选，撤销单张
}

// 行为：
// 从 previous* 字段恢复数据
// 清空 previous* 字段

13. 业务规则

┌──────────┬──────────────────────────────────────────────┐
│   规则   │                     说明                     │
├──────────┼──────────────────────────────────────────────┤

│ 撤销限制 │ 只能撤销最近一次修改                         │
├──────────┼──────────────────────────────────────────────┤
│ 图片去重 │ 通过 sha256 去重，相同图片只存一份           │
├──────────┼──────────────────────────────────────────────┤
│ 并发限制 │ 同一形象同时只能有一个生成任务               │
