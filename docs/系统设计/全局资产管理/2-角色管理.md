# 角色管理

## API 接口

```
├── 👤 角色管理
│   ├── GET    /characters                  获取角色列表 (支持 folderId 筛选)
│   ├── POST   /characters                  创建角色设定
│   ├── GET    /characters/[characterId]    获取单个角色
│   ├── PATCH  /characters/[characterId]    更新角色
│   └── DELETE /characters/[characterId]    删除角色
```

---

## 创建角色的两种模式

### 1. 文本描述模式

```
输入角色名 → 选择艺术风格 → 输入/生成描述词 → 提交 → 后台生成图片
```

**流程：**
1. 用户输入角色名（如"张三"）
2. 选择艺术风格（如"美漫风格"）
3. 输入描述词，或者点击"AI 设计"自动生成描述词
4. 提交后，后台异步生成图片

### 2. 参考图模式

```
上传参考图 → 选择生成方式 → 提交 → 生成图片
                             │
                             ├── 直接生成：模仿参考图风格
                             └── 提取提示词：AI 分析参考图生成描述词
```

---

## 创建角色流程详解

| 步骤 | 实际操作 |
|------|---------|
| 1 | 创建角色元数据（名字、风格） |
| 2 | 填写文本描述词 |
| 3 | 可选：上传参考图 |
| 4 | 提交后，异步生成图片 |

**注意：** 图片是异步生成的，创建时不会立即有图片。

---

## 命名说明

由于 AI 生成工具的特点，"创建角色"实际上做的是：

> 创建一个待生成角色的设定条目，图片是后续异步生成的。

### 传统流程 vs AI 工具流程

| 类型 | 流程 |
|------|------|
| 传统流程 | 画师画原画 → 确定形象 → 完善设定 |
| AI 工具流程 | 写设定/提示词 → AI 生成多版本 → 选最好的一张 → 再修改迭代 |

### 命名建议

| 当前命名 | 建议改为 |
|---------|---------|
| 创建角色 | 创建角色设定 |
| 创建场景 | 创建场景设定 |
| 角色形象 | 角色图片版本 |

## 数据模型

```prisma
model GlobalCharacter {
  id          String   @id @default(uuid())
  name        String                    // 角色名
  style       String?                   // 艺术风格
  description String?                   // 文本描述词
  folderId    String?                   // 所属文件夹
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  // 关联
  images      CharacterImage[]          // 多版本图片
}

model CharacterImage {
  id           String   @id @default(uuid())
  characterId  String
  mediaId      String                   // 关联 MediaObject
  prompt       String?                  // 生成时使用的提示词
  referenceUrl String?                  // 参考图 URL（如果有）
  status       ImageStatus              // pending | generating | completed | failed
  createdAt    DateTime @default(now())

  enum ImageStatus {
    pending
    generating
    completed
    failed
  }
}
```

## 异步生成任务

创建角色设定后，系统会创建异步任务生成图片：

```
创建角色设定 → 写入数据库 → 发布任务到 Redis Streams
                                    ↓
                              Worker 消费任务
                                    ↓
                              调用 AI Provider 生成图片
                                    ↓
                              上传到 MinIO → 更新状态
```
