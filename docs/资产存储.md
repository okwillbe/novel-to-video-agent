# 资产存储架构

这个项目采用 **元数据 + 媒体文件分离存储** 的设计：

```
┌─────────────────────────────────────────────────────────────┐
│                      资产存储架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────┐        ┌─────────────────────────┐   │
│   │    MySQL 8.0    │        │   MinIO (S3 兼容)       │   │
│   │   ─────────────  │        │   ───────────────────    │   │
│   │   元数据存储     │        │   媒体文件存储           │   │
│   │                 │        │                         │   │
│   │ • 资产基本信息   │        │ • 图片 (jpg/png/webp)   │   │
│   │ • 用户/项目信息  │        │ • 视频 (mp4/webm)       │   │
│   │ • 文件引用关系   │        │ • 音频 (mp3/wav)        │   │
│   └────────┬────────┘        └───────────┬─────────────┘   │
│            │                              │                 │
│            │         MediaObject          │                 │
│            │    ┌────────────────────┐    │                 │
│            └────┤ storageKey (索引)   ├────┘                 │
│                 │ sha256 (去重)       │                      │
│                 │ mimeType/size/...  │                      │
│                 └────────────────────┘                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

## 1. 元数据存储：MySQL

存储资产的结构化信息：

| 数据类型 | 说明 |
|---------|------|
| GlobalCharacter / NovelPromotionCharacter | 角色元信息 |
| GlobalLocation / NovelPromotionLocation | 场景元信息 |
| GlobalVoice | 音色元信息 |
| MediaObject | 媒体文件索引 |

### MediaObject 设计（关键）

```prisma
model MediaObject {
  id         String   @id @default(uuid())
  publicId   String   @unique          // 业务层唯一ID
  storageKey String   @unique          // MinIO 中的对象键
  sha256     String?                   // 文件哈希（去重用）
  mimeType   String?                   // MIME类型
  sizeBytes  BigInt?                   // 文件大小
  width      Int?                      // 图片宽度
  height     Int?                      // 图片高度
  durationMs Int?                      // 音视频时长
}
```

**设计亮点：**
- `sha256` 实现文件去重，相同内容只存一份
- `publicId` 与 `storageKey` 分离，业务层不暴露存储路径
- 宽高、时长等元信息缓存，避免重复获取

## 2. 媒体文件存储：MinIO（默认）

支持 3 种存储后端：

```typescript
type StorageType = 'minio' | 'local' | 'cos'
```

| 存储类型 | 说明 | 适用场景 |
|---------|------|---------|
| minio | S3 兼容对象存储（默认） | 生产环境、Docker 部署 |
| local | 本地文件系统 | 开发测试 |

### 默认配置

```yaml
STORAGE_TYPE: minio
MINIO_ENDPOINT: "http://minio:9000"
MINIO_BUCKET: "noveler"
MINIO_ACCESS_KEY: "minioadmin"
MINIO_SECRET_KEY: "minioadmin"
MINIO_FORCE_PATH_STYLE: "true"
```

## 3. 图片处理流程

```typescript
// 下载并上传图片时自动处理
export async function downloadAndUploadImage(imageUrl, key) {
  const sharp = await import('sharp')
  const buffer = await fetch(imageUrl).then(r => r.arrayBuffer())

  // 1. 转为 JPEG，初始质量 95
  let processed = await sharp(buffer)
    .jpeg({ quality: 95, mozjpeg: true })
    .toBuffer()

  // 2. 如果超过 10MB，降低质量直到满足要求
  while (processed.length > 10 * 1024 * 1024 && quality > 60) {
    quality -= 5
    processed = await sharp(buffer).jpeg({ quality }).toBuffer()
  }

  // 3. 上传到 MinIO
  return await uploadObject(processed, jpgKey)
}
```

## 5. 签名 URL 机制

访问私有文件需要签名 URL：

```typescript
// 本地存储：通过 API 代理
if (provider.kind === 'local') {
  return `/api/files/${encodeURIComponent(key)}`
}

// MinIO/COS：签名 URL（有时效性）
return `/api/storage/sign?key=${key}&expires=${expires}`
```

## 总结

| 层级 | 存储 | 内容 |
|-----|------|------|
| 元数据 | MySQL | 资产信息、引用关系、文件索引 |
| 媒体文件 | MinIO/Local/COS | 图片、视频、音频 |
| 去重机制 | sha256 哈希 | 相同文件只存一份 |
| 访问控制 | 签名 URL | 私有文件安全访问 |
